# Get Algarve

A Django-powered corporate website

## Setup Instructions

### 1. Create Virtual Environment

```bash
python -m venv venv
source venv/bin/activate  # On Windows: venv\Scripts\activate
```

### 2. Install Python Dependencies

```bash
pip install -r requirements.txt
```

### 3. Install Node Dependencies and Build CSS

```bash
cd node
npm install
npm run build:css
cd ..
```

### 4. Set Up Database

```bash
python manage.py migrate
python manage.py createsuperuser
```

### 5. Run Development Server

```bash
python manage.py runserver
```

Visit http://localhost:8000

## Google Cloud Storage (Production)

This project is configured to use Google Cloud Storage for media files in production with **domain-based folder organization**.

### How It Works

Files are automatically organized by domain in the GCS bucket:
```
your-bucket-name/
├── example-com/          # Files from example.com
│   ├── site_images/
│   └── uploads/
├── blog-example-com/     # Files from blog.example.com
│   └── site_images/
└── localhost/            # Files from local development
    └── site_images/
```

This allows multiple sites to share a single GCS bucket while keeping files isolated.

### Setup Instructions

1. **Create a Google Cloud Storage bucket:**
   ```bash
   gsutil mb -p your-project-id gs://your-bucket-name
   ```

2. **Create a service account:**
   - Go to Google Cloud Console → IAM & Admin → Service Accounts
   - Create a new service account with "Storage Object Admin" role
   - Download the JSON key file
   - **Important:** Add the JSON file to `.gitignore` (already configured)

3. **Configure environment variables:**
   Create a `.env` file (copy from `.env.example`) and set:

   **Option A - File Path (Local Development):**
   ```
   USE_GCS=true
   GS_BUCKET_NAME=your-bucket-name
   GS_PROJECT_ID=your-project-id
   GOOGLE_APPLICATION_CREDENTIALS=/path/to/service-account-key.json
   DEFAULT_DOMAIN=example-com
   ```

   **Option B - JSON String (Railway/Heroku):**
   ```
   USE_GCS=true
   GS_BUCKET_NAME=your-bucket-name
   GS_PROJECT_ID=your-project-id
   GCS_CREDENTIALS_JSON={"type":"service_account","project_id":"...","private_key":"..."}
   DEFAULT_DOMAIN=example-com
   ```

4. **Set bucket permissions (for public read access):**
   ```bash
   gsutil iam ch allUsers:objectViewer gs://your-bucket-name
   ```

### Development vs Production

- **Development (USE_GCS=false)**: Media files stored locally in `media/` directory
- **Production (USE_GCS=true)**: Media files stored in Google Cloud Storage bucket

All uploaded files (logos, site images, etc.) will automatically be stored in GCS when enabled.

### File URLs

When GCS is enabled, files are accessible at:
```
https://storage.googleapis.com/your-bucket-name/example-com/site_images/logo.png
```

The domain folder is automatically determined from the request's domain name.

## Maintenance Mode

This site includes a built-in maintenance mode feature that can be toggled from the admin panel.

### How to Use

1. **Enable Maintenance Mode:**
   - Go to the admin panel: http://localhost:8000/admin
   - Click on "Site Settings"
   - Check the "Maintenance mode" checkbox
   - Click "Save"

2. **What Happens:**
   - Regular visitors will see a user-friendly maintenance page
   - Staff users (logged into admin) can still access the site normally
   - Staff users will see a warning banner when maintenance mode is active
   - The maintenance page automatically retries every 30 seconds

3. **Disable Maintenance Mode:**
   - Uncheck the "Maintenance mode" checkbox in Site Settings
   - Visitors will immediately regain access to the site

### Use Cases
- Performing updates or migrations
- Making major content changes
- Troubleshooting issues
- Scheduled maintenance windows
